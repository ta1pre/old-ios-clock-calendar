# Proceed Rules

UIプロトタイプ作業で次の段階へ進むための条件です。

## Stages

### Stage 1: UI Direction

次へ進む条件:

- 古いiPhone / iPad / iPad mini前提が明記されている
- iOS 12世代とUIKit前提が明記されている
- 縦向きと横向きの両対応が明記されている
- 横向きは左に時計表示、右にカレンダーが明記されている
- 縦向きは上に時計表示、下にカレンダーが明記されている
- 追加しない機能が明記されている

### Stage 2: Screen Requirements

次へ進む条件:

- 対象端末サイズの基準がある
- アナログ時計の正円維持が要件化されている
- 月間カレンダーの7列 x 最大6行が要件化されている
- 今日の日付の強調表示が要件化されている
- 設定画面の項目が要件化されている
- 小さい画面での破綻ポイントが整理されている

### Stage 3: UI Prototype Implementation

次へ進む条件:

- Must項目に未解決の違反がない
- iOS 12非対応APIの疑いがない
- 左右レイアウトの主役が時計表示とカレンダーになっている
- 実機またはシミュレータで確認すべき項目がTODOとして明記されている
- 実機署名の問題とUI互換性の問題が切り分けられている

## Must Gate

以下が1つでも未解決なら、次へ進まない。

- SwiftUI前提になっている
- iOS 13以降専用APIが必須になっている
- 縦向きだけ、または横向きだけに偏っている
- 時計表示またはカレンダー以外の機能が主役になっている
- 今日の日付の強調表示がない
- iPhone縦向きまたは横向きで主要表示が収まらない
- ユーザーへの説明なしにスコープを広げている

## Score

点数は補助として使う。

- 90点以上: 次へ進んでよい
- 80から89点: 改善点を明記してから進む
- 79点以下: 先に方針を見直す

ただし、Must Gate違反がある場合は点数に関係なく進まない。
