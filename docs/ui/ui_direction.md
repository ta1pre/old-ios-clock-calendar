# UI Direction

## Purpose

このフェーズでは、UIKitで作る置き時計アプリの最小構成を固める。

対象は古いiPhone / iPad / iPad mini。iOS 12世代の端末を基準にし、縦向きと横向きの両方で使えるようにする。

## Primary Screen

最初に作る画面は、縦横両対応のフルスクリーン画面。

表示するものは以下だけ。

1. 左側: アナログ時計
2. 右側: 月間カレンダー
3. カレンダー内の今日の日付を青い丸で表示
4. 右上のシンプルな `i` ボタンで設定画面を開く

## Design Principles

- 追加機能を増やさない
- 横向きは左右2カラム、縦向きは上下2段構成を基本にする
- 時計表示を少し強めに扱う
- 古いiPhoneでは時計表示をやや優先し、カレンダーは読み切れる最小サイズを保つ
- iPad / iPad miniでは余白を増やし、画面が間延びしないようにする
- 設定導線は右上の小さな1ボタンに限定する
- 古い端末で重くなる装飾を避ける

## Layout Direction

基本比率:

- iPhone landscape: 左 45% / 右 55%
- iPad mini landscape: 左 42% / 右 58%
- iPad landscape: 左 40% / 右 60%
- iPhone portrait: 上 40% / 下 60%
- iPad mini portrait: 上 38% / 下 62%
- iPad portrait: 上 36% / 下 64%

アナログ時計は正円を維持する。カレンダーは7列 x 最大6行の月間表示にする。

## Visual Direction

- 背景は暗めを標準にする
- アナログ時計は数字あり、目盛りあり、赤い秒針あり
- ライトモードも持つ
- カレンダーは通常日を控えめにし、今日だけ青い丸で強調する
- 日曜は赤、土曜は青にする
- 眩しすぎる白を主役にしない
- 設定ボタンは小さく、薄めにして主表示を邪魔しない

## Out of Scope for Now

- アラーム
- 通知
- 設定保存以外の状態保存
- テーマ切り替え
- インタラクティブな表示切り替え
- 予定表示
- 祝日表示
- 天気
- メモ
- App Store提出設定
- PWA/WebView実装
- 複雑なアニメーション
- 実時刻の精密な更新ロジック
- 多言語対応

## Questions to Resolve Later

- 今日の日付の青い強調をどの程度強くするか
- iPhone横向きでカレンダー文字が小さすぎないか
- iPhone縦向きでカレンダーの縦密度が高すぎないか
