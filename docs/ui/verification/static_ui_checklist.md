# Static UI Prototype Checklist

このチェックは、左の時計表示 + 右の月間カレンダー + 最小設定画面を持つUIKitアプリ用です。

## Target Sizes

- 568 x 320 pt: 最小iPhone横向きリスク確認
- 667 x 375 pt: iPhone 6横向き基準
- 320 x 568 pt: 最小iPhone縦向きリスク確認
- 375 x 667 pt: iPhone 6縦向き基準
- 1024 x 768 pt: iPad mini横向き基準
- 1024 x 768 pt: iPad横向き基準
- 768 x 1024 pt: iPad mini縦向き基準
- 768 x 1024 pt: iPad縦向き基準

## Must Check

- 左に時計表示がある
- 右に月間カレンダーがある
- 今日の日付だけが強調されている
- ダーク / ライトが切り替わる
- アナログ案では丸時計が正円を維持している
- アナログ案では数字と目盛りが読みやすい
- 赤い秒針が見える
- カレンダーが7列 x 最大6行で収まっている
- 右上の `i` ボタンで設定画面が開く
- 設定画面で `Light` / `Dark` が切り替わる
- 設定画面で `Sunday` / `Monday` / `Saturday` が切り替わる
- 横向きで画面が破綻していない
- 縦向きで画面が破綻していない

## Device-Specific Check

### 568 x 320 pt

- カレンダーの日付が読める
- 時計表示が小さくなりすぎない
- 今日の強調が強すぎない
- 設定画面が窮屈すぎない

### 320 x 568 pt

- 時計が上側で正円を保てる
- カレンダーが下側で6週分収まる
- 設定ボタンが主表示の邪魔をしない

### 667 x 375 pt

- 左右のバランスが自然
- アナログ案では針が読める
- アナログ案では数字が読める
- カレンダーの月タイトルと曜日が読める

### 375 x 667 pt

- 上下のバランスが自然
- カレンダーの日付が縦向きでも読める
- 設定ボタンが小さすぎず、目立ちすぎない

### 1024 x 768 pt

- 余白が広すぎて間延びしない
- 時計表示が必要以上に大きくなりすぎない
- カレンダーが右側で読みやすい
- 設定画面の2グループが読みやすい

### 768 x 1024 pt

- 上下に積んでも間延びしない
- カレンダーが下側で読みやすい
- 設定画面の2グループが読みやすい

## Current Environment Note

Xcode simulator build verification is available. Physical iOS 12 device verification is still a separate final check.
